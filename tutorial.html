1 Visão Geral do Sistema

O GOOD AÇAÍ ERP é um sistema de gestão empresarial completo, projetado para lojas de açaí com operação de delivery (iFood, 99Food) e atendimento presencial.

📊
Dashboard KPIs de faturamento, pedidos, despesas e lucro em tempo real
📦
Pedidos Gestão completa: receber, preparar, entregar e concluir
💰
Financeiro Contas a pagar/receber, filtros por período, exportação CSV
👥
Equipe Cadastro de funcionários com status ativo/inativo
Design: Estilo "Quiet Luxury" com paleta Dourado (#FFD700) e Preto (#1A1A1A). Tipografia premium com DM Serif Display + DM Sans.

2 Acessando o ERP

Modo Desenvolvimento

  1. Abra o terminal na pasta do projeto
  2. Execute: npm run dev
  3. Acesse no navegador: http://localhost:5183/

Modo Produção

  1. Execute: npm run build
  2. Abra o arquivo dist/index.html no navegador
💡 Dica: O sistema inicia sempre com dados zerados (localStorage limpo) para segurança. Use o botão "Executar Simulação" para popular com dados de demonstração.

3 Perfil e Papéis

O sistema possui dois papéis com visões diferentes:

👔 Dono (Proprietário)

  • Acesso total: Dashboard, Pedidos, Financeiro e Equipe
  • Visualiza KPIs de Faturamento e Lucro
  • Gerencia contas a pagar/receber
  • Exporta relatórios CSV

👷 Funcionário

  • Acesso ao painel de Pedidos e Equipe apenas
  • Foco na operação: receber, preparar e despachar pedidos
  • Não vê dados financeiros sensíveis
Como trocar de perfil: Clique no card do Pedro (canto superior esquerdo da sidebar) → selecione "Dashboard (Dono)" ou "Painel de Preparo (Funcionário)".

O badge amarelo pulsante (💛) no avatar do Pedro aparece quando há contas urgentes (vencendo em 48h).

5 KPIs — Indicadores

📈
Faturamento Soma de todas as entradas (vendas + receitas)
📦
Pedidos Total de pedidos recebidos no período
📉
Despesas (Vermelho) Contas a pagar + saídas. Sempre em vermelho #DC143C e negrito
💚
Lucro (Verde / Vermelho) Faturamento - Despesas. Verde #008000 se positivo (+), Vermelho se negativo (-)
⚠ Regra fixa: Despesas são sempre vermelhas. Lucro positivo é sempre verde com sinal +. Lucro negativo é sempre vermelho com sinal -. Essas cores são travadas no CSS.

6 Gestão de Pedidos

Criando Pedidos

  • + Pedido iFood — Simula pedido via iFood (ícone vermelho)
  • + Pedido 99Food — Simula pedido via 99Food (ícone amarelo)
  • ▶ Monitorar — Ativa monitoramento automático (pedidos a cada 30s)

Fluxo de Status (Kanban)

Cada pedido passa por 4 etapas. Clique "Avançar" para mover:

  1. 🟡 Pendente → Pedido recebido, aguardando preparo
  2. 🟠 Em Preparo → Na cozinha (timer de 15 min ativado)
  3. 🔵 Saiu para Entrega → Pedido despachado
  4. 🟢 Concluído → Entrega confirmada

Ações no Card do Pedido

  • Avançar — Move para o próximo status
  • 🖨 Imprimir — Abre comanda para impressão/PDF
  • 🗑 Remover — Exclui o pedido
🔔 Alerta Sonoro: Cada novo pedido emite um beep (880Hz, 300ms) via AudioContext para chamar atenção do preparador.

Busca de Pedidos

Use o campo "Buscar pedido..." para filtrar por nome do cliente, canal ou produto.

Tabela "Últimos Pedidos"

Abaixo dos cards, uma tabela lista os últimos pedidos com: Cliente, Valor, Status e Data.

Clique no botão "Exportar CSV" para baixar um relatório completo em formato .csv compatível com Excel. O arquivo respeita o filtro de período selecionado.

10 Impressão de Comandas

Como Imprimir

  1. Vá em Pedidos
  2. No card do pedido, clique no botão 🖨 Imprimir
  3. Um popup abre com a comanda formatada
  4. O diálogo de impressão aparece automaticamente
  5. Selecione a impressora (ou "Salvar como PDF" para teste)

Formato da Comanda

  • Largura: 280px (80mm térmico)
  • Fonte: Monospace para alinhamento perfeito
  • Conteúdo: Nome da loja, nº do pedido, canal, cliente, valor e data/hora
  • CSS @page { size: 80mm auto } para impressoras térmicas
💡 Sem impressora? No diálogo de impressão, selecione "Salvar como PDF" ou "Microsoft Print to PDF" para gerar um arquivo PDF da comanda.

12 Versão Mobile

O sistema é totalmente responsivo para telas menores que 768px:

  • Sidebar escondida → substituída por barra de navegação inferior (bottom nav)
  • KPIs empilhados em colunas 2x2
  • Tabelas transformadas em cards empilhados
  • Dashboard grid em coluna única
  • Padding inferior de 100px para não cobrir o bottom nav
💡 Teste Mobile: No navegador, pressione F12 → clique no ícone de celular (Device Toolbar) → selecione um dispositivo como "iPhone 14" ou "Samsung Galaxy".

13 Informações Técnicas

Stack Tecnológica

  • Frontend: HTML5, CSS3 (Grid/Flexbox), JavaScript Vanilla
  • Build: Vite v5.4.21
  • Gráficos: Chart.js (CDN)
  • Fontes: Google Fonts — DM Serif Display + DM Sans
  • Áudio: Web Audio API (AudioContext) para beep de pedidos
  • Armazenamento: localStorage (chave: good-acai-erp-state)

Comandos do Projeto

npm run devInicia servidor de desenvolvimento
npm run buildGera build de produção em /dist
npm run previewPreview do build de produção

  • style.css — Estilos CSS do tema Quiet Luxury
  • comanda-teste.html — Comanda de teste para impressão/PDF
  • dist/ — Build de produção otimizada
  • package.json — Configuração npm + Vite
